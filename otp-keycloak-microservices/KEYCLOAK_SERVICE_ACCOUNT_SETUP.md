# Keycloak Service Account Setup

## Security Best Practice: Use Service Account Instead of Admin

**NEVER use admin credentials in your application code!** This document explains how to create a dedicated service account with limited privileges for your microservices.

---

## Why Service Account?

Using admin credentials in code is a **critical security risk**:
- ❌ Full access to Keycloak (can delete realms, modify system settings)
- ❌ If code is exposed, complete security breach
- ❌ Violates principle of least privilege
- ❌ No audit trail for service actions

**Service Account Benefits:**
- ✅ Limited permissions (only what's needed)
- ✅ Separate credentials from admin
- ✅ Better audit trail
- ✅ Can be rotated independently
- ✅ If compromised, limited blast radius

## Setup Instructions

### Step 1: Login to Keycloak Admin Console

1. Navigate to: `http://localhost:8180/admin`
2. Login with admin credentials:
   - Username: `admin`
   - Password: `admin`

### Step 2: Create Service User

1. Go to **Clients** → Select **backend-service** client
2. Go to **Credentials** tab → Copy the **Client Secret** (save for later)
3. Go to **myrealm** (your realm) → **Users**
4. Click **Add User** button

**User Details:**
```
Username: service-user-manager
Email: service@yourcompany.com
Email Verified: ON
Enabled: ON
```

5. Click **Create**

### Step 3: Set Password for Service User

1. After creating user, go to **Credentials** tab
2. Click **Set Password**
3. Enter a strong password:
   ```
   Password: [Use a strong password, e.g., generated by password manager]
   Temporary: OFF (Important!)
   ```
4. Click **Save**

### Step 4: Assign Required Roles

1. Stay in the user's page, go to **Role Mappings** tab
2. Under **Client Roles**, select: `realm-management`
3. Assign the following roles from **Available Roles** to **Assigned Roles**:
   - `view-users` - Read user information
   - `manage-users` - Create and update users
   - `query-users` - Search users

**These are the MINIMUM roles needed for the Identity Auth Service to:**
- Search for existing users
- Create new users during registration
- Update user attributes

### Step 5: Verify Permissions

Test that the service user has the correct permissions:

```bash
# Get access token for service user
curl -X POST http://localhost:8180/realms/myrealm/protocol/openid-connect/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password" \
  -d "client_id=backend-service" \
  -d "client_secret=YOUR_CLIENT_SECRET" \
  -d "username=service-user-manager" \
  -d "password=YOUR_SERVICE_PASSWORD"

# Use the access token to test user management
# (Replace ACCESS_TOKEN with the token from above)
curl -X GET http://localhost:8180/admin/realms/myrealm/users \
  -H "Authorization: Bearer ACCESS_TOKEN"
```

If you get a 200 response with users list, permissions are correct!

## Application Configuration

### Step 1: Update `identity-auth-service/application.yml`

**Before (INSECURE):**
```yaml
keycloak:
  server-url: http://localhost:8180
  realm: myrealm
  client-id: backend-service
  client-secret: your-client-secret
  admin-username: admin          # ❌ NEVER DO THIS
  admin-password: admin           # ❌ NEVER DO THIS
```

**After (SECURE):**
```yaml
keycloak:
  server-url: ${KEYCLOAK_SERVER_URL:http://localhost:8180}
  realm: ${KEYCLOAK_REALM:myrealm}
  client-id: ${KEYCLOAK_CLIENT_ID:backend-service}
  client-secret: ${KEYCLOAK_CLIENT_SECRET}
  admin-username: ${KEYCLOAK_SERVICE_USERNAME}
  admin-password: ${KEYCLOAK_SERVICE_PASSWORD}
```

### Step 2: Create `.env` File (Local Development)

Create `.env` file in project root (add to `.gitignore`!):

```bash
# Keycloak Configuration
KEYCLOAK_SERVER_URL=http://localhost:8180
KEYCLOAK_REALM=myrealm
KEYCLOAK_CLIENT_ID=backend-service
KEYCLOAK_CLIENT_SECRET=your-backend-service-client-secret
KEYCLOAK_SERVICE_USERNAME=service-user-manager
KEYCLOAK_SERVICE_PASSWORD=your-strong-service-password
```

### Step 3: Load Environment Variables

**Option A: Using IntelliJ IDEA / IDE**
1. Run → Edit Configurations
2. Environment Variables → Add all variables from `.env`

**Option B: Using Terminal (Linux/Mac)**
```bash
export KEYCLOAK_CLIENT_SECRET="your-client-secret"
export KEYCLOAK_SERVICE_USERNAME="service-user-manager"
export KEYCLOAK_SERVICE_PASSWORD="your-service-password"

# Then run the service
cd identity-auth-service
mvn spring-boot:run
```

**Option C: Using Terminal (Windows)**
```cmd
set KEYCLOAK_CLIENT_SECRET=your-client-secret
set KEYCLOAK_SERVICE_USERNAME=service-user-manager
set KEYCLOAK_SERVICE_PASSWORD=your-service-password

cd identity-auth-service
mvn spring-boot:run
```

**Option D: Using Spring Boot Maven Plugin**
```bash
mvn spring-boot:run \
  -Dspring-boot.run.arguments="\
    --keycloak.client-secret=your-client-secret \
    --keycloak.admin-username=service-user-manager \
    --keycloak.admin-password=your-service-password"
```

## Production Deployment

### Using Docker

**docker-compose.yml:**
```yaml
services:
  identity-auth-service:
    image: identity-auth-service:latest
    environment:
      - KEYCLOAK_SERVER_URL=https://keycloak.yourcompany.com
      - KEYCLOAK_REALM=production-realm
      - KEYCLOAK_CLIENT_ID=backend-service
      - KEYCLOAK_CLIENT_SECRET=${KEYCLOAK_CLIENT_SECRET}
      - KEYCLOAK_SERVICE_USERNAME=${KEYCLOAK_SERVICE_USERNAME}
      - KEYCLOAK_SERVICE_PASSWORD=${KEYCLOAK_SERVICE_PASSWORD}
```

### Using Kubernetes Secrets

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: keycloak-credentials
type: Opaque
stringData:
  client-secret: your-client-secret
  service-username: service-user-manager
  service-password: your-service-password
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: identity-auth-service
spec:
  template:
    spec:
      containers:
      - name: identity-auth-service
        image: identity-auth-service:latest
        env:
        - name: KEYCLOAK_CLIENT_SECRET
          valueFrom:
            secretKeyRef:
              name: keycloak-credentials
              key: client-secret
        - name: KEYCLOAK_SERVICE_USERNAME
          valueFrom:
            secretKeyRef:
              name: keycloak-credentials
              key: service-username
        - name: KEYCLOAK_SERVICE_PASSWORD
          valueFrom:
            secretKeyRef:
              name: keycloak-credentials
              key: service-password
```

## Security Checklist

- [ ] Service account created with limited roles (not admin)
- [ ] Strong password set for service account
- [ ] Credentials stored in environment variables (not in code)
- [ ] `.env` file added to `.gitignore`
- [ ] Production secrets managed via secrets manager (AWS Secrets Manager, HashiCorp Vault, etc.)
- [ ] Regular password rotation policy established
- [ ] Audit logs enabled for service account actions

## Troubleshooting

### Issue: "Insufficient permissions" error

**Solution:** Verify service account has required roles:
- Go to Keycloak → Users → service-user-manager → Role Mappings
- Check that `realm-management` client roles are assigned

### Issue: "Invalid credentials" error

**Solution:**
1. Verify environment variables are set correctly
2. Check that service user password matches what's configured
3. Ensure password is not temporary

### Issue: Can't login with service account

**Solution:**
1. Check user is enabled in Keycloak
2. Verify email is verified (if required by realm)
3. Check password is not expired or temporary

## Additional Security Recommendations

### 1. Use Vault for Secret Management (Production)

Instead of environment variables, use HashiCorp Vault or cloud provider secrets:

**AWS Secrets Manager:**
```java
@Bean
public Keycloak keycloak(SecretsManagerClient secretsClient) {
    String secret = getSecret(secretsClient, "prod/keycloak/service-account");
    // Parse and use secret
}
```

### 2. Rotate Credentials Regularly

Set up a rotation policy (every 90 days):
1. Create new service user
2. Update application configuration
3. Deploy with new credentials
4. Disable old service user after verification

### 3. Use Client Credentials Grant (Alternative)

For services that only need to get tokens (not manage users), use client credentials:

```yaml
# No username/password needed
keycloak:
  client-id: backend-service
  client-secret: ${KEYCLOAK_CLIENT_SECRET}
```

```java
// Get service account token
public String getServiceToken() {
    MultiValueMap<String, String> formData = new LinkedMultiValueMap<>();
    formData.add("grant_type", "client_credentials");
    formData.add("client_id", clientId);
    formData.add("client_secret", clientSecret);

    // Make token request
}
```

## Migration from Admin Credentials

If you're currently using admin credentials:

1. ✅ Follow steps above to create service account
2. ✅ Update application.yml
3. ✅ Set environment variables
4. ✅ Test application functionality
5. ✅ Deploy to all environments
6. ✅ Remove hardcoded admin credentials from all code
7. ✅ Change admin password in Keycloak
8. ✅ Audit code repository for any remaining hardcoded credentials
